# Architecture Decision Records (ADR)

Ce dossier contient les **Architecture Decision Records** du projet CAP ALI. Un ADR est un document court qui capture le contexte, la décision et les conséquences d'un choix technique important.

## Index des ADR

| #    | Titre                                        | Statut   | Date       |
| ---- | -------------------------------------------- | -------- | ---------- |
| 0001 | NuxtJS 3 comme framework full-stack          | Accepté  | 2026-05-05 |
| 0002 | PostgreSQL + Drizzle ORM                     | Accepté  | 2026-05-05 |
| 0003 | Architecture monolithique répliquée          | Accepté  | 2026-05-05 |
| 0004 | JWT + OAuth Google pour l'authentification   | Accepté  | 2026-05-05 |
| 0005 | API REST plutôt que GraphQL                  | Accepté  | 2026-05-05 |

## Modèle pour un nouvel ADR

Voir le [template](./template.md) pour créer un nouveau record.

## Statuts possibles

- **Proposé** — En cours de discussion
- **Accepté** — Validé par l'équipe
- **Déprécié** — Remplacé par un ADR plus récent
- **Remplacé** — Voir l'ADR de remplacement
